# Service Fulfillment Process for DevHassan Studio

**Date:** 2026-01-24
**Business Name:** DevHassan Studio
**Website:** https://www.devhassan.space

---

## 1. Offer Description

DevHassan Studio provides professional custom web development services, categorized into three distinct tiers:

- **Starter Plan:** Single-page applications and landing pages.
- **Professional Plan:** Multi-page websites with CMS integration and advanced animation.
- **Enterprise Plan:** Complex custom web applications, e-commerce solutions, and full-stack development.

These are **non-tangible services** delivered digitally through secure code repositories, direct transfer of intellectual property (source code), and live server deployments.

## 2. Fulfillment Workflow

Our service delivery follows a strict, transparent 5-step process:

### Step 1: Purchase & Agreement

- The client selects a service tier (e.g., Enterprise) and completes the secure payment via Whop.
- A transaction ID is generated, and a receipt is automatically sent to the client.

### Step 2: Immediate Onboarding

- Upon successful payment, the client is automatically redirected to our confirmed payment page (`https://www.devhassan.space/payment-redirect`).
- Within **24 hours**, our project management team contacts the client via the email associated with the transaction to initiate the discovery phase (gathering requirements, design assets, and technical specifications).

### Step 3: Development Phase & Milestones

- A private **GitHub repository** is created for the client's project to ensure version control and security.
- The client is provided with regular updates corresponding to key milestones:
  - **Alpha:** Initial design implementation and layout.
  - **Beta:** Functional features and interactivity.
  - **Release Candidate:** Final polish and content integration.

### Step 4: Final Delivery

- Upon approval of the Release Candidate:
  - The **Source Code** is transferred to the client (via GitHub invite or secure zip archive).
  - The website is **Deployed** to the client's live production environment (e.g., Vercel, Netlify, or private VPS).

### Step 5: Post-Launch Support

- The service includes a guaranteed 30-day support period starting from the date of deployment to resolve any bugs or minor adjustments.

## 3. Where Customers Access the Product

Since our product is a custom-built digital asset, access is provided in two ways:

1.  **Live Access:** The fully functional website effectively "lives" at the client's public domain URL.
2.  **Asset Access:** The client receives full ownership of the code, accessible via their private GitHub repository or secure file transfer.
